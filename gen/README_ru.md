# Генератор языковых привязок

В папке gen расположены файлы, предназначенные для создания языковых привязок.
Основу генерации составляют файлы данных генератора в формате
[S-выражений](https://ru.wikipedia.org/wiki/S-%D0%B2%D1%8B%D1%80%D0%B0%D0%B6%D0%B5%D0%BD%D0%B8%D0%B5).

Исторически проект libBeresta вырос из необходимости вывода *.pdf файлов в проекте,
написанном на Common Lisp, поэтому выбор формата был логичным следствием работы.

По мере накопления списка функций и генерации из них языковой привязки для [ECL](https://ecl.common-lisp.dev/)
стало понятно, что такое представление очень удобно, причем как для редактирования,
так и для дальнейшего использования в качестве основы генерации.

Файлы данных генератора (`*.lsp`) напрямую используются для генерации привязки ECL
(см. [brst-binding-ecl](https://github.com/libBeresta/brst-binding-ecl)), однако использование
S-выражений может быть не очень удобно в языках, отличных от семейства Lisp.

Для генерации привязок с помощью других языков из файлов данных с S-выражениями созданы
файлы формата JSON и YAML, содержащими ту же самую информацию, что и файлы данных, но в
форматах JSON и YAML соответственно.

## Доработка файлов JSON и YAML
По мере развития библиотеки файлы данных генератора будут развиваться и изменяться.
Для поддержания файлов JSON и YAML в актуальном состоянии в сборочную систему (CMake)
добавлен механизм обновления этих файлов.

Необходимо при настройке проекта указать параметр `-DLIBBRST_JSON_YAML=ON`, это добавит
цели сборки `json` и `yaml` в проект.

После этого будет доступна возможность вызова `make json` и `make yaml`, пересоздающие
файлы форматов JSON и YAML соответственно.

Цель `yaml` зависит от цели `json`.

Для корректной работы механизма необходимо установить утилиту
[`yq`](https://github.com/mikefarah/yq) версии не ниже 4.52 и сделать её
доступной в переменной `$PATH`.
